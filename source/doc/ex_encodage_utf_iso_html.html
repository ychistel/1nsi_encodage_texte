L'encodage de caractères

Une page web doit déclarer l'encodage de caractères utilisé pour que le navigateur s'adapte et affiche correctement les contenus. Si la norme aujourd'hui est utf-8, il existe encore de nombreux sites qui ne l'utilisent pas! Du coup, certains caractères peuvant s'afficher de façon incorrecte à cause d'un problème de déclaration non reconnue par le navigateur.

En général, il faut juste modifier cette déclaration dans le fichier pour rendre l'affichage lisible. Cela coûte 0 €.

Exercice

  1. Pouvez-vous reconnaitre les caractères mal affichés ?
  2. La dernière phrase "Cela coûte 0 €." est mal affichée. Sur combien d'octets sont encodés les caractères mal affichés en utf-8 ?
  3. Selon vous, quel est l'encodage utilisé et déclaré sur cette page web ?
  4. Certains navigateurs permettent de changer l'encodage de texte. Est-ce le cas pour votre navigateur ?
  5. Pour corriger l'erreur d'affichage, il faut éditer le fichier et modifier l'encodage. Faites la modification et vérifier que le navigateur affiche correctement la page.
