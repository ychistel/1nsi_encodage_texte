Tables d'encodage
=================

Nous allons nous intéresser à trois tables qui ont marqué l'évolution de
l'encodage des caractères. Avec un éditeur de texte, vous allez réaliser une
présentation des trois tables de caractères en suivant le plan donné ci-dessous.

Table de caractères ASCII
-------------------------

#. Que signifie l'acronyme ASCII ? Quel est son origine ? Combien de
   caractères contient-elle ?
#. Insérer dans votre document une image de la table ASCII.
#. Quel est le nombre de bits utilisés pour chaque caractère ? Donner une plage
   (intervalle) du premier au dernier caractère ASCII.
#. Á quoi correspondent les 32 premiers caractères de cette table ? Et le dernier.
#. Quels caractères correspondent aux codes hexadécimaux 6E, 53 et 49  ? Donner
   pour chaque caractère sa valeur binaire et décimale.
#. Quel est le principal problème de cette table de caractères pour notre langue ?

Tables ISO
----------

#. Que signifie l'acronyme ISO ?
#. Combien de tables contient la norme ISO-8859 ?
#. Combien de caractères peut contenir une table à la norme ISO-8859 ?
#. Quel est le nombre de bits utilisés par chaque caractère ? Donner une plage
   (intervalle) du premier au dernier caractère ISO-8859 non ASCII.
#. Quelles ont été les deux tables ISO-8859 utilisées en France ? Pourquoi ?


Unicode
-------

#. Quel est l'avantage de la norme Unicode sur la norme ISO ?
#. Qu'est-ce qu'un point de code dans la norme Unicode ? Comment se note-t-il ?
#. Quelles sont les points de code des caractères ASCII ?
#. Quel est le point de code associé au caractère È (alt+212) ? æ (alt+145) ? £
   ?
#. Quel est le caractère associé au point de code est U+20AC ?
#. UTF-8 encode en binaire les caractères à partir de leur point de code.
   Combien d'octets sont nécessaires pour l'encodage d'un caractère ?
